## 二、架构设计

### 1. 进度条跳转

**当前状态：**
- 对话模式：已实现（仅在有缓存音频时可用）
- 普通模式：未实现

**设计方案：**

**方案 A - 基于音频时长跳转（推荐）**
- 优点：精确、用户体验好
- 缺点：需要预先生成或计算音频时长
- 实现：
  - 对话模式：已有缓存音频，直接使用 `audio.currentTime`
  - 普通模式：预估每个 segment 的时长（字数 × 平均语速）

**方案 B - 基于 segment 索引跳转**
- 优点：实现简单
- 缺点：跳转不够精确（只能跳到句子开头）
- 实现：点击进度条 → 计算目标 segment → 停止当前播放 → 从目标 segment 开始

**选择：方案 A** - 用户期望时间精确跳转

### 2. 播放速度控制

**实现位置：**
- HTML5 Audio：`audio.playbackRate = speed`
- Web Speech API：`utterance.rate = speed`
- Qwen TTS：生成时指定速度参数

**UI 设计：**
```
控制条新增速度选择器：
┌────────────────────────────────────────────┐
│ ▶ ⏸ ⏹ | 进度条 | 🎵Cherry | 🎬1.0x | 💬教育 │
└────────────────────────────────────────────┘

```

**速度档位：**
- 0.5x（慢速）
- 0.75x（较慢）
- 1.0x（正常）- 默认
- 1.25x（较快）
- 1.5x（快速）
- 2.0x（极速）

### 3. 预设角色模板

**模板列表：**

| 模板名称 | 角色数 | 角色设定 | 音色分配 | 适用场景 |
|---------|--------|---------|---------|---------|
| 单人讲解 | 1 | 讲解者 | Ethan | 知识传授、教程 |
| 双人对话 | 2 | 对话者A、对话者B | Cherry, Serena | 轻松讨论、播客 |
| 三人课堂 | 3 | 讲师、好奇学生、批判学生 | Ethan, Cherry, Serena | 深度学习、课堂 |

**Prompt 调整：**

### 4. 对话风格选择

**风格定义：**

**正式风格：**
- 语言规范、逻辑严密
- 使用专业术语
- 适合学术、技术文档

**轻松风格：**
- 语言通俗、比喻生动
- 适当使用口语
- 适合科普、生活类内容

**幽默风格：**
- 加入幽默元素、俏皮话
- 使用类比和段子
- 适合娱乐、趣味内容

**实现方式：**
在 prompt 中添加风格指导语句。

## 五、技术实现要点

### 1. 进度条跳转实现

**对话模式（有缓存音频）：**
```typescript
handleProgressClick(e: MouseEvent) {
  const percentage = clickX / progressBarWidth;
  const targetTime = totalDuration * percentage;
  this.multiVoicePlayer.seekToTime(targetTime);
}
```

**普通模式（无缓存音频）：**
```typescript
handleProgressClick(e: MouseEvent) {
  const percentage = clickX / progressBarWidth;
  const targetIndex = Math.floor(totalSegments * percentage);
  await this.engineManager.stop();
  await this.engineManager.speakSegments(
    segments.slice(targetIndex)
  );
}
```

### 2. 播放速度实现

```typescript
// 对话模式
audio.playbackRate = speed;

// 普通模式（Web Speech）
utterance.rate = speed;

// Qwen TTS：在请求中添加 speed 参数
parameters: {
  speech_rate: speed  // -500 到 500
}
```
